# Eduard Admin Design

## Intent

The admin is an operator tool, not a landing page. The first screen must answer:

- Is the system running?
- What blocks sellability?
- Where do I change prices?

## Layout

- Put the work surface first: preview on the left, price controls on the right.
- Keep operational diagnostics below the work surface.
- Show only three top-level status cards: Betrieb, Proof, Feedback.
- Collapse rare settings such as mail, dealer details, signature, colors, and CSV.

## Components

- Use compact cards only for metrics and repeated review items.
- Use tables for editable product groups and rules.
- Keep primary actions visible: Speichern, Digest senden, Gruppe hinzufügen, Regel hinzufügen.
- Keep internal keys ASCII, but user-facing labels UTF-8 German.

## Copy

- Use direct German labels.
- Prefer business words over technical words.
- Avoid explaining features inside the UI unless an action is blocked.
